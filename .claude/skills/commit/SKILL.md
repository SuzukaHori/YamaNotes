## 手順

### 1. ブランチ確認

`git branch --show-current` で現在のブランチを確認する。

- **main ブランチにいる場合**: 作業内容に合った feature ブランチ名をユーザーに提案し、作成してから進める
- **feature ブランチにいる場合**: そのまま続行

### 2. 変更内容の把握

以下を実行して変更の全体像を掴む。

```bash
git status
git diff
git diff --cached
```

### 3. Lint の実行と自動修正

`bin/lint` を実行する。

- **エラーなし**: そのまま次のステップへ
- **自動修正可能なエラーあり**: 以下のコマンドで修正する
  ```bash
  bundle exec rubocop -a          # Ruby
  npm run fix                     # JS/CSS (prettier --write)
  ```
  修正後、再度 `bin/lint` を実行してクリーンになったことを確認する。
- **自動修正できないエラーが残った場合**: エラー内容をユーザーに報告し、コミットを続けるか中断するか確認する。ユーザーが「このままコミット」と言った場合のみ続行する。

### 4. コミット内容の提案

以下をユーザーに提示して承認を得る。承認なしにコミットしてはならない。

- ステージングするファイルの一覧
- コミットメッセージ（下記の形式）

#### コミットメッセージの形式

```
<type>: <変更内容を一言で表す日本語の説明>

<変更の背景（ユーザーから情報がある場合のみ）>

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

`type` の例: `feat`, `fix`, `chore`, `refactor`, `docs`, `test`, `style`

### 5. コミット実行

ユーザーの承認後、ステージングしてコミットする。

```bash
git add <承認されたファイル>
git commit -m "..."
```

### 6. プッシュの確認

コミット完了後、「プッシュしますか？」とユーザーに確認する。承認された場合のみ `git push` を実行する。

### 7. PR 作成

プッシュ完了後、`gh pr view` でこのブランチの PR がすでに存在するか確認する。PR がない場合のみ `create-pr` スキルを起動してPRを作成する。
