# GPS トラッキング要件定義書

## 概要

Walk 中の GPS 座標を一定間隔で収集・保存し、歩いた経路と正確な距離を地図上に表示する。
Walk ごとにトラッキングの ON/OFF を設定できる。

---

## 背景・解決したい問題

### 現状の課題

- 歩いた距離が駅間の固定値による**推測値**であり、正確ではない
- 歩いた経路が不明なため、正確な距離や経路を知るには iPhone のヘルスケアやルートヒストリーなど外部アプリが必要になる
- 競合アプリには経路記録機能が存在する

### この機能で実現したいこと

- 歩いた経路と正確な距離を YamaNotes 内で完結して提供する
- 歩き終わった後、walk の show ページのスクリーンショットを友人などに共有できる

### メインの利用シーン

歩き終わった後の**振り返り**がメイン。リアルタイムの現在地確認は副次的。

---

## 要件

### 1. トラッキング設定

- Walk ごとにトラッキングの ON/OFF を設定できる（デフォルト: OFF）
- 設定箇所: Walk の new ページ（作成時）と edit ページ（作成後）

### 2. GPS 座標の収集・送信・保存

- トラッキング ON の Walk の show ページを開いている間、ブラウザが GPS 座標を定期収集する
- 送信失敗時はサイレントに無視する
- 送信後は経路表示できるよう保存される

### 3. 地図表示

- トラッキング ON の場合のみ、walk の show ページの地図に GPS 軌跡・現在地マーカーを表示する
- 公開ページには表示しない

#### 既存の赤いライン（到着済み駅を繋ぐ線）との関係（未決定）

以下のいずれかから実装時に決定する：

- **案 A**: GPS 軌跡で赤いラインを置き換える（駅間直線ラインを非表示にする）
- **案 B**: 駅間直線ライン（赤）と GPS 軌跡を両方表示する

### 4. 距離表示

- トラッキング ON の場合: GPS 軌跡から計算した正確な距離を表示する
- トラッキング OFF の場合: 従来通り推測距離（駅間固定値の合計）を表示する
